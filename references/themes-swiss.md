# 瑞士國際主義風格 · 主題色預設（Swiss Themes）

4 套基於瑞士國際主義風格（Swiss Style）的高反差配色。**每套都遵循「高階灰白底 + 單一高飽和高亮色」的極簡原則**——這是瑞士風的靈魂，不允許混搭多個高亮色。

---

## 使用方法

1. 問使用者選哪套（或基於內容推薦一套）
2. 開啟 `assets/template-swiss.html` 的 `<style>` 塊
3. 找到開頭的 `:root{` 塊
4. **整體取代**標有「主題色」註釋的所有變數：`--paper` / `--paper-rgb` / `--ink` / `--ink-rgb` / `--grey-1` / `--grey-2` / `--grey-3` / `--accent` / `--accent-rgb` / `--accent-on`
5. 其他 CSS 都走 `var(--...)`，無需任何其他改動

---

## 🔵 克萊因藍 (IKB · International Klein Blue)

**適合**：通用場合、商業釋出、AI/科技產品、設計領域分享。最經典的瑞士風配色，絕不出錯。
**調性**：純白底 + IKB 克萊因藍，極致冷靜、理性、有學術感，像 Helvetica Forever 或 Massimo Vignelli 的作品集。

```css
--paper:#fafaf8;
--paper-rgb:250,250,248;
--ink:#0a0a0a;
--ink-rgb:10,10,10;
--grey-1:#f0f0ee;
--grey-2:#d4d4d2;
--grey-3:#737373;
--accent:#002FA7;
--accent-rgb:0,47,167;
--accent-on:#ffffff;
```

**使用要點**：
- IKB 是高飽和深藍，在大色塊（如 `.accent-block`）上極有視覺衝擊
- KPI 數字加 `.accent` 類用藍色，但不要滿屏藍——IKB 一旦氾濫就掉檔
- 推薦配合 `dark` 主題頁交替使用，黑底高亮 IKB 同樣高階

---

## 🟡 檸檬黃 (Lemon · Cadmium Yellow)

**適合**：年輕、運動、零售、消費品、活力主題、Y2K 復古設計。
**調性**：淺米白底 + 檸檬黃，鮮亮、有活力、警示感強，像 IKEA 或 Beck Design 的視覺語言。

```css
--paper:#fafaf8;
--paper-rgb:250,250,248;
--ink:#0a0a0a;
--ink-rgb:10,10,10;
--grey-1:#f0f0ee;
--grey-2:#d4d4d2;
--grey-3:#737373;
--accent:#FFD500;
--accent-rgb:255,213,0;
--accent-on:#0a0a0a;
```

**使用要點**：
- 檸檬黃屬於淺色高飽和，**`.accent-on` 必須用純黑**（不是白）才能保證可讀性
- 黃色色塊上不要放白字——會糊掉
- 檸檬黃做單字元高亮（`.mark` / `.underline-accent`）效果最強

---

## 🟢 檸檬綠 (Lemon Green · Highlighter Green)

**適合**：生態、可持續、健康、新興科技、Z 世代品牌、AI 創業專案。
**調性**：淺米白底 + 螢光檸檬綠，有未來感、年輕、當代，像 Acne Studios 或 Off-White 的視覺。

```css
--paper:#fafaf8;
--paper-rgb:250,250,248;
--ink:#0a0a0a;
--ink-rgb:10,10,10;
--grey-1:#f0f0ee;
--grey-2:#d4d4d2;
--grey-3:#737373;
--accent:#C5E803;
--accent-rgb:197,232,3;
--accent-on:#0a0a0a;
```

**使用要點**：
- 螢光綠和黃色一樣屬於淺色，**`.accent-on` 必須用純黑**
- 螢幕顯色比印刷漂亮，適合演講投影場景
- 推薦用於「新興技術」、「未來」主題

---

## 🟠 安全橙 (Safety Orange)

**適合**：工業、警示、運動、施工、汽車工業、技術釋出會的「警告/重點」頁。
**調性**：淺米白底 + 安全橙，工業感、緊迫感、視覺錨點感，像 Saul Bass 海報或 Highway Gothic 標識系統。

```css
--paper:#fafaf8;
--paper-rgb:250,250,248;
--ink:#0a0a0a;
--ink-rgb:10,10,10;
--grey-1:#f0f0ee;
--grey-2:#d4d4d2;
--grey-3:#737373;
--accent:#FF6B35;
--accent-rgb:255,107,53;
--accent-on:#ffffff;
```

**使用要點**：
- 橙色介於淺色和深色之間，**白字勉強能讀但建議加粗**（`font-weight:600` 以上）
- 工業感強，適合涉及「警告」、「決策」、「轉折點」的內容
- 不建議整頁 `.accent` 模式，橙色滿屏會過於刺眼，做區域性高亮即可

## 轉場原則

- **一份 deck 只用一套主題**，不要中途換 accent 色
- 灰階變數（`--grey-1/2/3`）在 4 套主題裡完全相同，無需調整
- WebGL 網格背景會自動讀取 `--accent` 變數，翻頁時滑鼠附近會偷渡一抹高亮色
- 選定主題後，可以在 chrome 文案裡用一個相關詞強化語義（如 IKB 配 `International / Helvetica` ，檸檬黃配 `Active / Living`）

---

## ❌ 不要做的事

- ❌ **不允許混搭**（例如 IKB 藍 + 檸檬黃同時出現作高亮）——徹底違反瑞士風「單一錨點色」原則
- ❌ **不允許使用者自定義任意 hex 值**——委婉拒絕，展示 4 套預設讓選
- ❌ **不要改灰階變數**——`--paper` / `--grey-1/2/3` / `--ink` 跨主題統一，只換 accent
- ❌ **不要用漸層**——瑞士風拒絕任何漸層，所有色塊必須純色
- ❌ **不要給 accent 加陰影 / 圓角 / 透明度**——直角、純色、不透明，這是瑞士風的硬規則

---

## 關於灰階（跨主題統一）

| 變數 | 值 | 用途 |
|---|---|---|
| `--paper` | `#fafaf8` | 主底色（極淺暖白） |
| `--grey-1` | `#f0f0ee` | 淺灰底（用於 `.grey-block` / 區塊底） |
| `--grey-2` | `#d4d4d2` | 中灰（分割線、border） |
| `--grey-3` | `#737373` | 暗灰（輔助文字 / meta） |
| `--ink` | `#0a0a0a` | 文字主色（近黑） |

這套灰階是經過校色的「高階灰」，在任何 accent 色下都不搶戲。**不要**改成純白（`#fff`）或純黑（`#000`）——會損失瑞士風的「克制」質感。

---

選定主題後，告訴使用者：「用 🔵 克萊因藍 / 🟡 檸檬黃 ...」 並在 deck 專案記錄裡備註，方便後續迭代時保持一致。
