# GPT-M 2.0 配圖提示詞

用於 Codex 環境下為本 skill 產生 PPT 配圖。提示詞只負責定基調，不要寫成長篇說明。先判斷圖片落位和比例，再選取型別。

## 通用規則

- 先判斷當前 deck 風格：風格 A = 電子雜誌 × 電子墨水；風格 B = 瑞士國際主義 / Swiss Style
- 風格 A 基調：電子雜誌 × 電子墨水，克制、真實、留白充足，適合橫向網頁 PPT
- 風格 B 基調：Swiss International Typographic Style，12/16 列網格、Helvetica/Inter 氣質、單一高飽和 accent、直角純色、髮絲線、極大留白
- 資訊圖、圖表、截圖再設計中的文字語言必須跟隨使用者語言：中文 deck 用中文，英文 deck 用英文
- 不產生卡通、3D、霓虹科技感、SaaS 範本感、過度裝飾或假 logo
- 圖片要給標題或正文留出可疊加空間，不要滿屏堆細節
- 同一頁或同一組圖片必須使用同一比例、同一視覺縮放、同一邊距密度
- 配圖是嵌入 PPT 的素材，不是一張獨立 slide：不要產生頁首、頁尾、頁碼、標題列、角標、署名、裝飾邊框或 slide chrome
- 產生後儲存到 `images/`，命名為 `{頁號}-{語義}.{ext}`

## 比例選取

| 用途 | 推薦比例 | HTML 落位 |
|------|---------|-----------|
| 章節封面 / 全螢幕主視覺 | 16:9 | `.frame-img.r-16x9` 或 hero 背景參考 |
| 瑞士風頂部橫幅 / Image Hero | 16:9 或 21:9 | P22 頂圖 cover / `.frame-img.r-21x9` |
| 左文右圖主圖 | 16:10 或 4:3 | `.frame-img.r-16x10` / `.frame-img.r-4x3` |
| 資訊圖 / 系統關係圖 | 16:9 或 16:10 | 原始截圖用 `.fit-contain`；按槽位重產生則用 `.frame-img.r-16x9` / `.frame-img.r-16x10` 鋪滿 |
| 截圖再設計 / UI 情景圖 | 16:10 或 21:9 | 原始截圖用 `.fit-contain`；重產生到 S15/S16 時用 `.frame-img.r-21x9` 鋪滿 |
| 圖文混排小圖 | 3:2 或 3:4 | `.frame-img.r-3x2` / `.frame-img.r-3x4` |
| 圖片網格 | 統一橫圖 | `.frame-img.h-22` / `.frame-img.h-26` |
| 小型面板組 | 統一橫圖 | `.frame-img.h-16` / `.frame-img.h-18` |

資訊圖和截圖再設計如果來自不可控原始素材，優先用 `fit-contain`，避免文字被裁切；如果是 GPT-M 2.0 按槽位重新產生，必須產生同槽位比例並鋪滿容器，不要讓小圖漂在白框裡。紀實照片優先用預設 `cover`，保持畫面張力。

## 圖片標準化策略

### A. 先選目標槽位

不要先產生圖片再硬塞進頁面。先決定圖片落位：

1. 主視覺：16:9
2. 左文右圖：16:10 或 4:3
3. 資訊圖/截圖再設計：16:9 或 16:10，並使用 `fit-contain`
4. 多圖網格/面板組：統一高度類，同一組內禁止混用高度

### B. 使用者原始圖片/截圖的處理

原始截圖比例通常不可控，不要直接作為最終視覺標準。按下面順序處理：

1. 如果原圖內容需要保真，先讀 `screenshot-framing.md`，用 CleanShot X 式程式化適配：目標比例畫布 + 風格化背景 + 截圖等比縮放 + 語義化 padding/alignment
2. 如果原圖比例接近目標槽位，直接放入統一 `.frame-img` 中，用 `cover` 或 `fit-contain`
3. 如果一張 UI 圖被拉成巨長條，拆成 2-3 張同尺寸區域性面板；每個面板使用同一高度類
4. 如果原圖過高、過窄、過長且無法透過適配解決，再用「截圖再設計 / UI 情景圖」重新產生到目標比例
5. 如果必須保留原圖，用 `fit-contain` 放進統一 frame，接受留白，不要裁掉關鍵文字

### C. 產生提示詞字尾

每個配圖提示詞最後都補一句規格約束：

```text
輸出必須是[16:9/16:10/4:3/3:2]橫向構圖，主體居中但保留邊距，畫面密度中等，與同組圖片保持相同視覺縮放和邊距。只保留核心圖形/畫面本身，不要產生頁首、頁尾、標題、頁碼、角標、署名、裝飾邊框、超長條、豎圖或不規則比例。
```

同一頁需要多張圖時，補一句：

```text
這是一組圖片中的一張，請保持與同組圖片相同的畫面比例、元素大小、邊距、線條粗細和標註密度。
```

## 風格 B：瑞士國際主義配圖規則

當 deck 選取 `assets/template-swiss.html` / `layouts-swiss.md` 時，優先使用下面這組提示詞。它們和 GPT-M 2.0 配套，目標是產生能直接放進原始登記版式的圖片槽位，尤其是 S22 頂部橫幅、S15/S16 多圖網格。

### Swiss 配圖硬規則

- 視覺錨點：International Typographic Style / Swiss modernism / Helvetica / Josef Müller-Brockmann / Massimo Vignelli
- 構圖：嚴格 12/16 列網格、非對稱留白、靠左對齊、髮絲線、直角模組
- 色彩：只使用黑、白、灰和**一個**主題 accent（預設 IKB 藍；如果使用者選檸檬黃/綠/安全橙，就取代為對應 accent）
- 禁止：漸層、陰影、圓角、玻璃擬態、霓虹、3D、卡通、SaaS 範本感、偽 logo、裝飾邊框
- 圖片內部不要產生 PPT 外殼：不要頁首、頁尾、頁碼、標題列、角標、署名、外框
- UI / 資訊圖文字必須短，保持中文/英文語言一致；真實照片儘量不要帶文字
- 先確定版式槽位再產生圖片：單張大圖用 `s22-hero-21x9`；多圖格用 `s15-grid-21x9` 或 `s16-brief-21x9`
- 21:9 圖片必須讓核心主體落在中央 70% 安全區，四周留白；不要把人臉、關鍵節點或 UI 文字貼邊

### Swiss 型別 1：紀實照片 / 案例主圖

用於 S22 Image Hero，增加真實場景錨點。

```text
產生一張 21:9 超寬橫向紀實攝影配圖，主題是：[頁面概念]。風格是 Swiss editorial documentary：高對比、低飽和、冷靜克制、真實辦公/城市/產品使用場景，構圖有大量負空間，主體位於中央 70% 安全區，適合放入瑞士國際主義 PPT 的頂部橫幅。不要 AI 機器人、科幻介面、商業擺拍、logo、水印或文字。只保留核心照片本身，不要頁首、頁尾、標題、頁碼、角標、署名、裝飾邊框或 PPT 外殼。
```

### Swiss 型別 2：資訊圖 / 系統關係圖

用於解釋概念、架構、流程、資料與表現分離等抽象內容。

```text
產生一張橫向 Swiss Style 資訊圖，解釋：[概念/流程/系統關係]。使用 Helvetica/Inter 氣質的無襯線短標籤、12/16 列網格、直角模組、1px 髮絲線、黑白灰和一個 [IKB 藍/檸檬黃/檸檬綠/安全橙] accent。圖中文字使用[中文/英文]，每個標籤不超過 8 個字/詞。不要漸層、陰影、圓角、3D、卡通、霓虹或 SaaS 範本感。輸出比例為[21:9/16:10]，主體居中但保留大留白。只保留核心資訊圖本身，不要頁首、頁尾、標題、頁碼、角標、署名、裝飾邊框或 PPT 外殼。
```

### Swiss 型別 3：截圖再設計 / UI 情景圖

用於把截圖、工作區、程式碼、dashboard 重繪成統一 Swiss 風視覺。

